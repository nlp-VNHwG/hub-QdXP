BERT 文本分类与实体识别的关系及 Loss

关系：
两者共享同一个 BERT 编码器，区别仅在于输出层如何利用编码器的输出：
文本分类：只取输入序列第一个特殊标记 [CLS] 的输出向量。这个向量汇聚了整句话的语义信息，用于判断整句的类别。
实体识别：取序列中每一个 Token 对应的输出向量。每个向量独立预测该位置字的标签，属于序列标注任务。

使用的 Loss：
文本分类：交叉熵损失。
    计算对象：1 个 [CLS] 向量对比 1 个句子标签。
实体识别：逐 Token 交叉熵损失。
    计算对象：N 个 Token 向量对比 N 个字标签，通常对所有有效位置的 Loss 求平均。

直接相加的坏处

直接简单相加会导致严重的训练不平衡，主要原因如下：

数量级差异：
    seq_loss 是基于句子数量计算的。
    token_loss 是基于字符或词数量计算的。
    后果：token_loss 的数值通常比 seq_loss 大几十倍。在反向传播时，梯度主要由实体识别任务主导，文本分类任务的梯度微乎其微，导致分类模型几乎学不到东西。
收敛速度不一致：
    如果两个任务难度不同，简单的任务会迅速收敛，而难的任务 Loss 依然很大。此时总 Loss 完全由难的任务决定，可能导致简单任务的性能退化。

如何处理训练不平衡？

核心目标是让两个任务对总 Loss 的贡献权重相当。常用处理方法：

方法一：静态加权
手动引入超参数来平衡量级：
 Loss = lambda_1 cdot seq_loss + lambda_2 cdot token_loss 
操作技巧：
    由于 Token 数量远多于句子数量，通常设置 lambda_1 大于 lambda_2。
    或者，将 token_loss 除以平均序列长度，使其在数值上与 seq_loss 处于同一量级，然后设两个系数均为 1。

方法二：动态加权
让模型在训练过程中自动学习权重，无需人工调参：
不确定度加权：为每个任务引入一个可学习的参数，代表任务的不确定度，根据 Loss 的变化动态调整权重。任务越难学，权重自动越高。
GradNorm：监控两个任务的梯度范数，强制调整权重，使得两个任务的梯度下降速度保持一致。

总结建议：
最简单有效的方法是将 token_loss 除以当前 Batch 的平均序列长度，然后再与 seq_loss 相加。
