DeepSeek-V3 继承了 V2 首创的 MLA（多头潜在注意力，Multi-head Latent Attention） 和 DeepSeekMoE 核心架构，但在模型规模、底层训练框架、网络算法设计以及后训练策略上进行了全方位的升级。以下是具体的对比总结：
1. 模型规模与数据量的全面扩大
•	DeepSeek-V2: 总参数量 236B（2360亿），单 token 激活参数量 21B；预训练数据量为 8.1T Tokens。
•	DeepSeek-V3: 总参数量跃升至 671B（6710亿），单 token 激活参数量增加到 37B；预训练数据量大幅扩展至 14.8T 高质量 Tokens。V3 在显著扩大规模的同时，由于架构优势，依然保持了极高的训练和推理经济性。
2. 训练目标的创新：引入 MTP (Multi-Token Prediction)
•	V2: 采用传统语言模型的单词预测（Next-Token Prediction）。
•	V3: 创新性地引入了 MTP（多词预测） 机制。在预测当前词的同时，模型会在独立模块中预测后续的多个词。这一改进不仅让模型获得了更密集的训练信号和更强的上下文规划能力，在推理阶段，MTP 模块还可以直接用作投机解码（Speculative Decoding）的草稿模型，最高可将生成速度提升一倍。
3. MoE 负载均衡策略的根本性改变
•	V2: 像大多数传统 MoE 模型一样，依赖辅助损失函数（Auxiliary Loss）来强制让各个专家（Experts）平均分担计算量。但辅助损失往往会干扰模型主任务的训练表现。
•	V3: 提出了 无辅助损失的负载均衡（Auxiliary-loss-free Load Balancing）。V3 通过在训练过程中动态调整路由器的“偏置项（Bias）”来控制 token 分发。这一机制在保证专家负载均衡的同时，彻底消除了辅助损失对核心性能的负面影响。
4. 极致的底层工程优化 (FP8 与 DualPipe)
•	训练精度：V3 实现了全面且稳定的大规模 FP8 混合精度训练（V2 主要基于标准的 BF16）。V3 团队通过细粒度的量化策略克服了低精度训练中的数值不稳定问题，极大地降低了显存占用并成倍提升了计算吞吐量。
•	并行通信：针对 MoE 模型在集群训练中跨节点通信开销大的痛点，V3 放弃了传统的流水线并行，设计了全新的 DualPipe 并行算法。它实现了前向/反向计算与跨节点全对全（All-to-All）通信的完美重叠（Overlap），将 GPU 利用率推向了极限。
5. 后训练（Post-Training）与对齐策略的进化
•	知识蒸馏：V3 的后训练阶段大量吸纳了来自其内部强大的推理模型（DeepSeek-R1 系列）的推理轨迹数据。通过将长链条的复杂推理能力蒸馏到标准 V3 模型中，其数学和代码能力得到了史诗级增强。
•	强化学习（RL）：V3 更加成熟地应用了 GRPO（群组相对策略优化） 算法，并且结合了基于规则的奖励（针对数学/代码的确定性反馈）和基于模型的奖励机制，以极低的成本完成了高质量的人类偏好对齐。
总结： DeepSeek-V2 证明了“MLA + 细粒度 MoE”架构的理论可行性与高效性；而 DeepSeek-V3 则是通过“无辅助损失负载均衡 + 引入 MTP + 极限的 FP8 与 DualPipe 工程优化”，将这一架构的潜力榨取到了极致，成功以极低的训练算力成本打造出了比肩全球第一梯队闭源模型的产品。
