# 作业1答案：BERT多任务学习分析

## 问题1：BERT文本分类和实体识别的关系及Loss函数

### 关系分析

**1. 架构上的关系**
- **共享底层编码器**: 文本分类和实体识别都使用同一个BERT编码器
- **不同输出头**:
  - 文本分类: 使用 `pooler_output` (CLS token的输出,形状 [batch_size, 768])
  - 实体识别: 使用 `last_hidden_state` (每个token的输出,形状 [batch_size, seq_len, 768])

**2. 数据流向**
```python
# 从 model.py 第27-31行可以看到
bert_output = self.bert(input_ids, attention_mask, token_type_ids)
pooler_output = bert_output[1]  # 用于文本分类
token_output = bert_output[0]   # 用于实体识别
seq_output = self.sequence_classification(pooler_output)  # 文本分类输出
token_output = self.token_classification(token_output)     # 实体识别输出
```

**3. 互补性**

- **文本分类**: 判断整体意图（如"查询"、"预订"、"取消"）
- **实体识别**: 提取关键实体（如"许昌"→出发地，"中山"→目的地）
- 两者结合可以完整理解用户需求

### Loss函数

**两种任务都使用 CrossEntropyLoss**

```python
# 从 main.py 第48-50行
self.criterion = nn.CrossEntropyLoss()  # 第19行定义

seq_loss = self.criterion(seq_output, seq_label_ids)      # 文本分类loss
token_loss = self.criterion(active_logits, active_labels) # 实体识别loss
loss = seq_loss + token_loss
```

**具体说明**:
- **seq_loss**: 意图识别的交叉熵损失
  - 输入: `[batch_size, num_intent_classes]`
  - 目标: `[batch_size]` (意图标签)
  - 作用: 优化模型正确分类整体意图

- **token_loss**: 实体识别的交叉熵损失
  - 输入: `[batch_size * active_tokens, num_ner_classes]` (只计算有效token)
  - 目标: `[batch_size * active_tokens]` (BIO标签)
  - 作用: 优化模型正确识别每个token的实体类型

## 问题2：多任务训练的坏处及不平衡处理

### `loss = seq_loss + token_loss` 的坏处

**1. 梯度量级不一致**
```python
seq_loss ≈ 1.5  (25个意图类，概率分布较均匀)
token_loss ≈ 15 (序列长度32，62个BIO标签，累计误差大)
```
- token_loss通常比seq_loss大10-20倍
- 会导致模型主要优化实体识别，意图识别被忽略

**2. 损失值范围不同**
- seq_loss: 0~ln(25) ≈ 3.22 (25个意图类)
- token_loss: 0~ln(125) ≈ 4.83 (O + 2×62个标签)
- 不同范围的loss直接相加不合理

**3. 学习速度不同**
- token_loss下降更快，容易过拟合
- seq_loss可能还没有学到特征就被token_loss主导

**4. 负迁移风险**
- 某些任务的梯度可能对另一个任务有害
- 例如：实体识别的梯度可能损害意图分类

### 训练不平衡的处理方法

#### 方法1: 动态权重调整
```python
# 不确定性加权 (Kendall et al., 2018)
loss = (1/σ1²)*seq_loss + (1/σ2²)*token_loss + log(σ1*σ2)

# GradNorm (Chen et al., 2018)
loss = α*seq_loss + β*token_loss
# 通过让不同任务的梯度范数成比例来调整α,β

# 简单的梯度裁剪
if seq_loss > token_loss * 10:
    loss = seq_loss * 0.1 + token_loss
else:
    loss = seq_loss + token_loss
```

#### 方法2: 损失归一化（推荐）
```python
# L1归一化
total_loss = seq_loss + token_loss
loss = (seq_loss / total_loss) * seq_loss + (token_loss / total_loss) * token_loss

# L2归一化
loss = (seq_loss / torch.norm(seq_loss)) + (token_loss / torch.norm(token_loss))
```

#### 方法3: 固定权重
```python
# 根据验证集调参
loss = 0.5 * seq_loss + 0.5 * token_loss  # 均衡权重
loss = 0.3 * seq_loss + 0.7 * token_loss  # 侧重实体识别
```
