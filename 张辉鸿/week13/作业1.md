# DeepSeek-V3 与 DeepSeek-V2 核心改进对比

DeepSeek-V3 在架构、效率、性能和成本控制上实现了全面升级。下表总结了主要改进：

| 方面 | DeepSeek-V2 | DeepSeek-V3 |
| :--- | :--- | :--- |
| **总参数量 / 激活参数量** | 236B / 21B | 671B / 37B（总参数量约 V2 的 **3 倍**，激活量仅增 1.7 倍） |
| **MoE 专家设计** | Top-6 路由，激活 2 个专家 | Top-8 路由，激活 **4 个专家**（共 256 个专家） |
| **负载均衡** | 依赖辅助损失函数 | 首创 **无辅助损失策略**，性能更优 |
| **多 Token 预测 (MTP)** | 无 | **支持**，提升训练推理效率与上下文连贯性 |
| **训练数据类型** | BF16 | **FP8** 混合精度（首批成功应用的大型 MoE 模型） |
| **训练成本 / 数据量** | ~276 万美元 / 8.1T tokens | **~557.6 万美元** / **14.8T tokens**（成本远低于同规模模型） |
| **推理速度（生成吐字）** | ~20 TPS | **60 TPS**（提升 **3 倍**） |
| **首 Token 延迟** | 基准 | 降低 **32%** |
| **基准性能 (MMLU)** | 68.7% | **87.1%**（大幅超越） |
| **上下文窗口** | 64K tokens | **128K tokens** |
| **核心能力** | 成本优化，高并发服务 | 长文本理解，复杂推理（代码、数学），多模态（视觉/语言） |

## 核心架构：更大、更聪明、更高效

- **膨胀的模型，克制的激活**：总参数 671B，计算时仅激活 37B 参数，实现效用的最大化。
- **MoE 专家系统大升级**：
  - 专家数量从 160 个提升至 **256 个**。
  - 激活专家数增至 4 个，采用 sigmoid 门控函数。
  - 引入创新的 **无辅助损失负载均衡策略**，避免专家闲置或过载。
- **多 Token 预测 (MTP)**：单步预测多个未来 Token，缓解短视问题，提升数据与推理效率。

## 训练与效率：低成本，高性能，高能效比

- **FP8 混合精度训练**：首个在超大规模 MoE 模型上成功部署 FP8 训练的开源模型，显著降低显存占用。
- **极致的成本控制**：参数量增长 3 倍、数据量翻倍的前提下，训练成本（~557.6 万美元）仅约为 V2（~276 万美元）的两倍，成本效率极高。

##  性能与能力：推理提速，任务全能

- **推理性能飞跃**：生成吐字速度从 20 TPS → 60 TPS，首 Token 延迟降低 32%。
- **全面卓越的基准测试**：
  - MMLU：68.7% → **87.1%**
  - SuperGLUE：89.3 分（达到顶尖水平）
  - 代码生成、数学推理等任务均大幅提升。
- **处理超长文本**：上下文窗口从 64K → 128K tokens，可一次性处理如《三体》三部曲体量的文本。
- **开启多模态能力**：为处理视觉语言任务奠定架构基础，支持图文联合建模。
