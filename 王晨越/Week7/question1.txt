Q:bert 文本分类和 实体识别有什么关系，分别使用什么loss？
A:
关系：
	两者在 BERT 架构下主要是特征抽取层共享，输出层不同：
		文本分类 (Sequence Classification)： 是“多对一”的任务。取 BERT 输出的 [CLS] 向量（或对所有 token 做池化），送入 Linear 层，预测整句话的标签。
		实体识别 (NER)： 是“多对多”的任务（序列标注）。取 BERT 输出的每一个 Token 的向量，分别送入 Linear 层，预测每个词的标签（如 B-PER, I-PER 等）。
Loss 函数：
	文本分类： 通常使用 Cross Entropy Loss (交叉熵)。
	实体识别：
		简单做法：对每个 Token 使用 Cross Entropy Loss。
		进阶做法：在输出层后接 CRF (条件随机场)，使用 Negative Log-Likelihood (NLL) Loss，以学习标签间的转移约束（如 I 标签不能在 B 标签前）。


Q:多任务训练  loss = seq_loss + token_loss 有什么坏处，如果存在训练不平衡的情况，如何处理？
A:
直接相加 (loss = loss1 + loss2) 的坏处：
	1.量级不一致： 分类（1个 label）和 NER（序列长度个 label）的 Loss 量级可能差几十倍，导致模型只优化 Loss 大的任务。
	2.收敛速度不同： 一个任务可能很快收敛，另一个还在震荡，固定权重会互相干扰。
	3.梯度冲突： 不同任务的梯度方向可能相反，导致模型“左右为难”，甚至性能不如单任务。
如何处理训练不平衡：
	1.手动调整权重 (Weighted Sum)：
		1.loss = α * seq_loss + β * token_loss。根据经验或验证集表现手动调整α和β.
	2.动态权重调整 (Dynamic Weight Averaging)：
		1.不确定性加权: 通过引入可学习的参数 ，根据任务的同方差不确定性自动调整权重。
		2.GradNorm: 动态调整权重，使不同任务的梯度范数（梯度大小）保持在相近的量级。
	3.训练策略调整：
		1.分步训练： 先预训练 NER，再微调分类；或者交替训练。
		2.梯度裁剪/遮掩： 如果某个任务梯度过大，进行缩放，防止它带偏整个模型的参数更新。
